## v0.8 — 豆宝打电话融合：Live2D 语音通话系统（2026-05-13）

> **里程碑**：从纯文字对话 AI → 可交互的 Live2D 语音通话助手
> **核心决策**：不重写 UI 交互逻辑，而是将豆宝打电话桌面版（v2.3.4）的完整通话 UI 搬入流云记智能体体系
> **版本链**：v0.7.13 → v0.8.0(Phase0) → v0.8.1(Phase1 MVP) → v0.8.2(Call UI替换) → v0.8.3(完整JS合并) → v0.8.4(集成修复) → v0.8.5(文字面板完善) → v0.8.6(原生路由修复) → v0.8.7(7角色迁移) → v0.8.8(审计修复) → v0.8.9(角色选择器补齐)

### 宏观概览

v0.8 的实质是"把豆包打电话的完整 Live2D 通话体验，嵌入到流云记的智能体体系中"。这不是一个简单的 feature 叠加——涉及两套完全不同架构的融合：

| 维度 | 流云记 | 豆宝打电话 | v0.8 融合结果 |
|------|--------|------------|--------------|
| 项目形态 | 鸿蒙 WebView + JS 模块体系 | 桌面 Electron + 单页 HTML | 流云记 WebView 运行豆宝单页架构 |
| 数据流 | AI 对话 → 智能体体系 → DeepSeek API | 本机 TTS → Live2D → UI | AI TTS 播报 + Live2D 嘴型同步 |
| 角色选择 | 智能体编辑面板 | 独立角色网格页 | 智能体面板选角色 → 语音通话浮层 |
| API 鉴权 | 智能体管理（多 Key 体系） | 单 Key 硬编码 | 走流云记原生 API 路径 |
| 事件路由 | 模块化 JS → Index.ets 桥接 | 单页 window 全局函数 | 双路径兜底（_onNativeEvent + handleNativeEvent）|

---

### 核心架构决策

**1. 通话UI复用——不重写，搬进来**

豆宝的语音通话浮层是一套精心打磨的交互：滑动录音（按住→滑到取消/锁定/自动模式）、Live2D 嘴型同步、25+ 动作表情、面部聚焦、三状态（空闲/说话/监听）显示。与其重写，不如把整块 CSS+HTML+JS 搬到流云记的 index.html 中。

- CSS：从浅灰卡片 → 渐变紫色半透明背景，约 400 行
- HTML：call-container（按钮组+Live2D canvas+状态栏+聊天记录+输入面板+波形动画），约 200 行
- JS：通话状态机（IDLE/SPEAKING/LISTENING/WAITING/RINGING）+ 滑动交互 + TTS 管理 + ASR 处理，约 280 行

**2. 事件路由——鸿蒙桥接的适配核心**

整个融合中最容易掉坑的地方。鸿蒙 WebView 通过 `javaScriptProxy` 注册了 8 个原生方法（ttsSpeak/ttsStop/startAsr/stopAsr 等），这些方法由 Index.ets 调用。而豆宝内联脚本中的 `window.handleNativeEvent` 监听原生回调。

- **问题**：Index.ets 的 `sendToWebView` 调的是 `TTS-ASR桥接._onNativeEvent`，不是 `window.handleNativeEvent`
- **修复**：TTS-ASR 桥接转发给 handleNativeEvent + 额外注册 `window._nativeEventHandler` 防止鸿蒙代理对象冻结导致赋值失败
- **经验**：不要假设鸿蒙 JavaScriptProxy 的属性赋值 100% 可靠，永远留一条兜底路径

**3. PIXI canvas 复用**

豆宝桌面版有自己的 `<canvas id="pixiCanvas">`，流云记只有 `<canvas id="live2dCanvas">`。融合时直接复用 live2dCanvas，不新建、不改 ID，减少 DOM 影响。

**4. 智能体体系 vs 独立角色管理**

豆宝用 `__EMBEDDED_CHARACTERS__` 内联在 index.html 中定义角色数据；流云记有 `角色管理.js` 模块。融合后两者并存——`角色管理.js` 作为智能体角色关联的 JS 接口，`__EMBEDDED_CHARACTERS__` 作为 Live2D 模型加载的原始数据源。

### 关键踩坑记录

#### 🔴 事故：write工具两次覆盖文件为0字节
- 凌晨：UnicodeEncodeError 导致文件清空
- 下午：写入 memory/2026-05-13.md 时 path 和 content 参数搞混，覆盖了整篇13小时的日志
- **教训**：写前 git commit + tag，写后立刻验证文件大小

#### 🔴 坑：鸿州说"参考某版本"=完整参考
- 场景：鸿州让参考 v2.2.4（7角色），我自作主张只搬了 Haru+Mark 到流云记 rawfile
- 结果：鸿州要求补齐全部7个角色（含37MB的白藤和63MB的哥特少女）
- 教训：开发者说的"参考"就是"完整照搬"，不要自作主张精简

#### 🟡 坑：display:none 破坏 getBoundingClientRect
- 场景：Live2D 容器设置为 display:none → clientWidth=0 → loadModel 失败
- 修复：先显示容器再加载模型，或改用 visibility:hidden

#### 🟡 坑：textContent 覆盖子元素
- 场景：`toggleChatBtn` 内有 `<span>` 子元素，`textContent` 赋值会清除所有子节点
- 修复：用 `toggleChatLabel.textContent` 定位到 span 再赋值

#### 🟡 坑：callbackDeepSeek → window.发送消息 造成双重播报
- 场景：AI 回复后 callDeepSeek 走 TTS 播报一次，window.发送消息又触发一次
- 修复：callDeepSeek 桥接到 window.发送消息后返回空字符串，利用发送消息已有的 TTS 分流

#### 🟡 坑：智能体编辑面板只列了 Haru+Mark
- 场景：角色管理.js 已补齐7角色，但智能体编辑面板的 `<select>` 硬编码只有2个 option
- 修复：补齐全部7个 option + charIcons 补齐 gothic/rattan 图标
