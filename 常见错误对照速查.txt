# 常见错误对照速查表（爱助手项目 · V6.0）

> 最后更新：2026-05-03
> 用途：开发过程中快速排查问题，避免重复踩坑
> 说明：只记录实际遇到的问题，不含规划中未实现的功能

---

## 一、模块加载与初始化

| 编号 | 问题 | 现象 | 原因 | 解决方案 |
|------|------|------|------|----------|
| LD-01 | 模块加载顺序错误 | 控制台报 `xxx is not defined`，功能不可用 | `index.html` 中 JS 引入顺序不对，被依赖的模块还没加载 | 确保核心模块（设置管理→备忘录数据→备忘录管理器→渲染→UI）先加载 |
| LD-02 | 文件夹路径不匹配 | 所有 JS 404 | HTML 引用 `js模块/` 但实际文件夹名是 `js` | 统一路径，确认文件夹名和 HTML src 一致 |
| LD-03 | 文件名后缀不匹配 | 单个 JS 404 | HTML 引用 `xxxA8.js` 但磁盘文件是 `xxx.js` | 去掉多余后缀，统一命名 |

## 二、AI 工具调用

| 编号 | 问题 | 现象 | 原因 | 解决方案 |
|------|------|------|------|----------|
| AT-01 | 工具定义后不执行 | AI 返回 tool_calls 但无动作 | 四步漏了某步 | 逐一检查：①工具定义 ②名称映射 ③路由白名单 ④处理函数 |
| AT-02 | 流式调用不支持 tool_calls | 函数调用时卡住/无响应 | DeepSeek 流式 API 不返回 tool_calls | 启用函数调用时强制走非流式（对话管理.js） |
| AT-03 | 对话历史污染 | AI 持续拒绝执行工具 | 失败记录写入历史，AI 从中学习 | 清空对话历史，或删除失败记录 |
| AT-04 | API调用.js 括号陷阱 | 语法错误，整个文件报错 | 工具定义的 `function {` 不是块语句而是对象属性，闭合 `}` 与 `parameters` 同级 | 手动检查括号配对；禁用 sed 批量替换缩进 |
| AT-05 | sed 批量替换误改 | 大量不相关的行被修改 | `sed 's/^    },$/  },/'` 匹配了所有 4 空格缩进的 `},` | 用 Python 脚本精确替换，或逐条编辑 |

## 三、备忘录筛选与渲染

| 编号 | 问题 | 现象 | 原因 | 解决方案 |
|------|------|------|------|----------|
| MR-01 | 筛选角标数量不一致 | 标题显示 "(4条)" 但实际只有 3 条 | memoIds 包含不存在/已删除的 ID | 用实际数据源过滤后计算数量，不用 memoIds.length |
| MR-02 | AI筛选标题数量重复显示 | "(4条) (3条)" 同时出现 | AI 生成的标题自带数量标记，UI 又加了一个 | 用正则 `/\s*\(\d+条\)\s*$/` 剥离标题中的数量标记 |
| MR-03 | 清除AI筛选按钮点击无响应 | 点击 × 无任何反应 | 事件委托被上游 stopPropagation 阻断 | 在渲染函数中用 `onclick` 直接绑定，最可靠 |
| MR-04 | 变量 TDZ 导致函数中断 | 文件夹点击不生效，控制台报 `Cannot access 'X' before initialization` | `const 当前标签筛选` 在第 734 行声明，但第 695 行条件 `!当前标签筛选` 提前引用→TDZ→抛 `ReferenceError`，`渲染备忘录列表()` 中断，文件夹筛选不执行 | 所有状态变量的 `const` 声明统一放在函数开头
| MR-05 | 筛选分支未互斥 | AI筛选 + 基础筛选同时执行 | AI 分支后没有 return，基础筛选仍执行 | AI 分支设完筛选后直接 return |
| MR-06 | 文件夹筛选不显示子文件夹 | 点击父文件夹只显示本级 | 筛选逻辑未递归查询子文件夹 | 用 `_获取所有子文件夹名()` 递归查询所有后代 |
| MR-07 | 回收站计数用了全量 | deleted 视图数量 = 全部数量 | 走了 `数据源.length` 未过滤 `已删除===true` | deleted 用 `m.已删除===true`，favorite 用 `m.收藏===true&&!m.已删除` |
| MR-08 | 空状态一直显示 | 有数据时空状态仍可见 | 分组渲染分支漏了 `空状态.style.display='none'` | CSS 默认 `display:none`，分组分支补显隐逻辑 |

## 四、UI 交互与事件

| 编号 | 问题 | 现象 | 原因 | 解决方案 |
|------|------|------|------|----------|
| UI-01 | 事件委托失效 | 按钮点击无响应，无控制台日志 | document 级别事件被 stopPropagation 阻断 | 渲染函数内用 `onclick` 直接绑定 |
| UI-02 | 文件夹下拉菜单关不掉 | 点击外部菜单不收 | 打开时设了内联 `style.display='block'`，关闭只移除 CSS 类 | 删除冗余内联样式设置，只用 CSS 类控制 |
| UI-03 | 更多菜单点击无效 | 菜单项点击无反应 | HTML 缺 `data-action` 属性，事件委托读到 undefined | 确保每个菜单项都有 `data-action` |
| UI-04 | 左滑菜单误触 | 左滑后点击立即触发菜单项 | touchend 后没有忽略点击保护 | touchend 后设 200-300ms 忽略点击保护（Rule #SwipeNoRebound） |
| UI-05 | 内联样式覆盖 CSS | CSS 类不生效 | JS 设置了内联 `style.xxx`，优先级高于 CSS 类 | 删除 JS 中的内联样式设置，统一用 CSS 类控制 |
| UI-06 | 面板切换不刷新 | 从对话切回备忘录时视图残留 | 标签页切换函数未调用 `渲染备忘录列表()` | 切换到备忘录面板时调用 `window.渲染备忘录列表()` |

## 四-B、contenteditable 富文本交互

| 编号 | 问题 | 现象 | 原因 | 解决方案 |
|------|------|------|------|----------|
| CE-01 | 退格删不掉最后一个字 | contenteditable 中只剩1字时退格无效 | 浏览器尝试合并节点而非删字 | 手动 `textContent = ''`，保留空待办让用户继续输入 |
| CE-02 | 退格删整行太激进 | 新建待办后删默认文字时整个待办消失 | `length <= 1` 直接 remove 整行 | 三档逻辑：空/开头→删整行；1字→手动删字留空待办；多字→浏览器默认 |
| CE-03 | insertHTML位置错乱 | 待办插入到编辑区最前面 | 修改DOM后range失效，insertHTML插入位置错误 | 用 `range.deleteContents()` + `range.insertNode()` 替代 insertHTML |
| CE-04 | 截断textNode后range失效 | 先截断endNode再操作range，位置跑到最前面 | 截断textContent破坏了range的引用 | 不截断，先操作range再处理剩余文本（包装nextSibling） |
| CE-05 | 选中建待办后文字重复 | 选区后面的文字出现两份 | insertHTML替换选区但剩余文字还在原位，代码又建了div | deleteContents+insertNode后，包装nextSibling文本节点到div中 |
| CE-06 | innerHTML='' 清不掉placeholder | CSS `:empty::before` 不触发 | `innerHTML = ''` 留下空标签，不算 `:empty` | 改用 `textContent = ''` |
| CE-07 | execCommand字号累积 | 拖动滑块只生效一次 | 每次execCommand创建新节点，保存的选区指向旧节点失效 | 缓存创建的span元素，后续拖动直接修改缓存元素fontSize |

## 五、设置与配置

| 编号 | 问题 | 现象 | 原因 | 解决方案 |
|------|------|------|------|----------|
| SC-01 | 设置值 `||` 默认值无效 | 期望的默认值没生效 | localStorage 存的是 truthy 值，`||` 不触发 | 用 `Math.max(值 \|\| 默认值, 默认值)` 或显式判断 |
| SC-02 | localStorage 持久值优先于代码 | 修改代码默认值不生效 | 用户之前设置过，localStorage 值一直存在 | 测试时清 localStorage 或用新 key |

## 六、鸿蒙 WebView 适配

| 编号 | 问题 | 现象 | 原因 | 解决方案 |
|------|------|------|------|----------|
| HM-01 | jsBridge 注入时机不确定 | `window.jsBridge` undefined | WebView 未完全加载就调用 | 前端轮询等待（最多 20 次 × 100ms），超时降级 |
| HM-02 | WebView 内存泄漏 | 多次打开关闭后卡顿 | registerJavaScriptProxy 未注销 | onPageDestroy 中清理 |
| HM-03 | touch 事件与 click 冲突 | 移动端点击行为异常 | touch 和 click 同时触发 | touchend 设保护期，或统一用 touchend |

## 七、IndexedDB 与存储

| 编号 | 问题 | 现象 | 原因 | 解决方案 |
|------|------|------|------|----------|
| DB-01 | 两个数据源不一致 | 筛选结果和显示数量不匹配 | `window._备忘录数据`（内存缓存）和 `window.备忘录管理器.memos`（IndexedDB）不同步 | 统一使用 `window._备忘录数据源 \|\| window._备忘录数据`，优先 IndexedDB |
| DB-02 | IndexedDB 迁移中断 | 旧数据丢失或重复 | 迁移逻辑没有事务保护 | 迁移前备份，失败回滚 |
| DB-03 | 文件夹结构丢失 | 刷新后文件夹树为空 | localStorage 被清空 | 定期备份文件夹结构到 IndexedDB |

## 八、调试速查

### 控制台常用命令

```javascript
// 查看数据源
window.备忘录管理器.memos          // IndexedDB 数据
window._备忘录数据                  // 内存缓存
window._AI临时筛选                  // AI 筛选状态
window._当前文件夹()                // 当前文件夹
window._当前筛选()                  // 筛选模式

// 强制刷新
window.渲染备忘录列表()
window.渲染文件夹树()

// 清除 AI 筛选
window._设置AI临时筛选(null)
```

### 日志关键词

| 关键词 | 含义 |
|--------|------|
| `[🔍调试]` | 调试日志（可搜索定位） |
| `[渲染]` | 渲染相关 |

| `[发送消息]` | 对话发送 |

### 查找代码

```bash
# 查函数定义
grep -rn "function 函数名\|window.函数名" js模块/

# 查事件绑定
grep -rn "addEventListener\|onclick" js模块/

# 查全局变量
grep -rn "window\._\|window\.变量名" js模块/
```
