> 最后更新：2026-04-27

> 用途：减少修改踩坑，快速定位模块

---

## 一、项目概述

- **技术栈**：纯前端 HTML + JS（33 个模块扁平结构），无框架
- **运行环境**：鸿蒙 WebView（移动端），Chrome（开发调试）
- **API**：DeepSeek API（对话/函数调用）+ 百度千帆（联网搜索）
- **存储**：IndexedDB（备忘录持久化）+ localStorage（设置/状态）+ 文件系统（AI记忆/智能体配置）

## 三、模块依赖关系

### 数据流

```
初始化.js
  → 备忘录管理器.js (IndexedDB 初始化)
  → 备忘录数据.js (状态初始化)
  → 备忘录渲染.js (首次渲染)
  → 备忘录UI.js (事件绑定)
  → 对话管理.js (会话恢复)
```

### 备忘录操作流

```
用户操作
  → 备忘录UI.js (事件捕获)
  → 备忘录数据.js (状态更新)
  → 备忘录管理器.js (IndexedDB 写入)
  → 备忘录渲染.js (UI 更新)
```

### AI 工具调用流

```
用户对话
  → 对话管理.js (发送消息)
  → API调用.js (调用 DeepSeek)
  → DeepSeek 返回 tool_calls
  → API调用.js (处理工具调用)
  → 备忘录管理器.js / 备忘录数据.js (执行操作)
  → 备忘录渲染.js (UI 更新)
  → API调用.js (继续对话)
```

### 关键依赖图

```
备忘录渲染.js
  ← 依赖 → 备忘录数据.js (获取数据源、文件夹树、筛选状态)
  ← 依赖 → 备忘录管理器.js (IndexedDB 数据)

备忘录数据.js
  ← 依赖 → 备忘录管理器.js (数据源)
  ← 被依赖 → 备忘录渲染.js, 备忘录UI.js, API调用.js

API调用.js
  ← 依赖 → 备忘录管理器.js (AI 工具操作)
  ← 依赖 → 备忘录数据.js (文件夹、筛选状态)
  ← 依赖 → 对话管理.js (系统提示词)
```

## 五、常见踩坑问题及解决方案

### 1. AI 工具调用失败

**问题**：AI 返回 tool_calls 但不执行

**根因**：四步缺一
1. 工具定义（`API调用.js` 顶部）
2. 工具名称映射（`工具名称映射` 对象）
3. 路由白名单（`处理工具调用` switch 分支）
4. 处理函数

**解决方案**：新增工具时，四步全部检查

### 2. 流式不支持 tool_calls

**问题**：启用函数调用时，流式调用卡住

**根因**：DeepSeek 流式 API 不返回 tool_calls

**解决方案**：`对话管理.js` 中，启用函数调用时强制走非流式

### 3. 对话历史污染

**问题**：AI 拒绝执行工具，且持续拒绝

**根因**：失败记录写入对话历史，AI 从历史学习

**解决方案**：必要时清空对话历史，或删除失败记录

### 4. 设置覆盖陷阱

**问题**：`设置值 || 默认值` 无效

**根因**：localStorage 存的是 truthy 值（如 15），`||` 不触发

**解决方案**：用 `Math.max(设置值 || 默认值, 默认值)` 或显式判断

### 5. 事件委托失效

**问题**：按钮点击无响应

**根因**：
- 事件绑定在模块函数内，可能元素不存在时绑定
- `document.addEventListener` 可能被其他代码 `stopPropagation` 阻断

**解决方案**：
- 在渲染函数中绑定（元素一定存在）
- 使用 `onclick` 直接绑定（最可靠）
- 避免上游 `stopPropagation`

### 6. 变量 TDZ 问题

**问题**：引用变量显示 undefined 或报错

**根因**：`const/let` 声明在引用位置之后（TDZ）

**解决方案**：将变量声明移到函数开头

### 7. 文件夹筛选不显示子文件夹备忘录

**问题**：点击父文件夹不显示子文件夹的备忘录

**根因**：筛选逻辑未递归查询子文件夹

**解决方案**：使用 `_获取所有子文件夹名()` 递归查询

### 8. 计数显示不一致

**问题**：标题显示 "(4条)" 但实际只有 3 条

**根因**：AI 返回的 memoIds 可能包含不存在/已删除的 ID

**解决方案**：用实际数据源过滤后计算数量，剥离标题中的数量标记

### 9. contenteditable 退格不删最后一个字

**问题**：待办项只剩1个字时，退格无法删除

**根因**：浏览器在 contenteditable 中只剩1字时，退格行为不是删字而是尝试合并节点

**解决方案**：三档逻辑
- 空/光标在开头 → `todoItem.remove()` 删整行
- 只剩1字 → 手动 `todoText.textContent = ''`，保留空待办
- 多字 → 允许浏览器默认退格

### 10. insertHTML 位置错乱

**问题**：选中文字创建待办后，待办插入到编辑区最前面

**根因**：修改 DOM（截断 textNode）后 range 失效，`insertHTML` 插入位置错误

**解决方案**：用 `range.deleteContents()` + `range.insertNode()` 替代 `insertHTML`，不截断 textNode，操作 range 后再处理剩余文本

## 六、修改时注意事项

### 新增 AI 工具

1. 在 `API调用.js` 顶部添加工具定义（JSON Schema）
2. 在 `工具名称映射` 添加中英文映射
3. 在 `处理工具调用` switch 添加分支
4. 编写处理函数（参考现有结构）
5. 在 `对话管理.js` 的 `获取记忆管理规则()` 添加工具说明（可选）

### 新增备忘录功能

1. 数据层：`备忘录数据.js` 或 `备忘录管理器.js`
2. 渲染层：`备忘录渲染.js`
3. UI 层：`备忘录UI.js` 或 `备忘录事件.js`
4. 触发 `window.渲染备忘录列表()` 刷新

### 新增 UI 元素

1. HTML：`index.html`
2. CSS：`css/样式.css`
3. 事件：`备忘录UI.js`（事件委托）或渲染函数中 `onclick`

### 修改筛选逻辑

1. 状态：`备忘录数据.js` 的 getter/setter
2. 渲染：`备忘录渲染.js` 的 `渲染备忘录列表()` 筛选分支
3. 确保各筛选分支互斥（AI / 基础 / 文件夹 / 日期）
