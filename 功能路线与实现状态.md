## 七、AI 知识自主管理工具（V6.2 规划）

> 让智能体能主动利用备忘录管理自己的长期记忆——不依赖对话上下文的脆弱性。
> 通用工具集，任何智能体（学习教练、小说生成、异世界闯荡、代码助手…）都能复用。

### 设计思想

现有 AI 只能被动等用户发指令操作备忘录。V6.2 要赋予 AI **自主决定

自主决定"该记什么、从哪读、怎么用"——不依赖对话上下文的脆弱性。

### 核心工具集（通用，不限场景）

| 工具 | 功能 | 解决什么问题 |
|------|------|-------------|
| `note(key, value, folder)` | 记一条事实到指定文件夹 | AI 自主判断"这事该记" |
| `update_note(key, newValue)` | 更新已有记录 | 信息变更时修正，不产生碎片 |
| `append_to_note(key, addition)` | 追加内容（如逐章追加剧情） | 长流程分步记录 |
| `recall(keyword, folder?)` | 模糊检索相关记录 | 需要时自己翻记忆 |
| `read_context(folder)` | 读文件夹最新摘要快速恢复上下文 | 新会话快速"找回状态" |
| `summarize_records(folder)` | 把文件夹所有记录压缩成一份摘要 | 防止信息碎片化 |
| `verify_consistency(topic)` | 检查 topic 相关记录是否有矛盾 | 防 OOC / 防遗忘 |
| `identify_gaps(topic)` | 分析哪些设定信息还不完整 | 引导用户补充设定 |

### 应用场景举例

| 场景 | AI 怎么做 |
|------|----------|
| **异世界闯荡** | 开局读 `异世界设定` → 每推进一段剧情记到 `剧情档案` → 角色成长记 `角色日志` |
| **生成小说** | 查 `剧情档案` 获前情 → 生成新章节 → 写入新备忘录 → 建摘要 → 循环 |
| **学习教练** | 每次辅导记 `薄弱点` → 下次先 `recall` 直接切入上次没讲透的地方 |
| **代码助手** | 项目结构记 `架构笔记` → 决策记 `技术选型` → 历史查询不走对话 |
