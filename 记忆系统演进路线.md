## 阶段 v1.5 — 最小语义检索接入（P1.5）

**目标：** 在用户记忆条数膨胀前（≥100条），完成语义检索的轻量预埋

### 具体工作

| # | 任务 | 预计工时 | 说明 |
|---|---|---|---|
| 1 | Transformers.js 集成 | 半天 | 引入 all-MiniLM-L6-v2（~23MB），首次加载缓存 |
| 2 | 写入时编码向量 | 半天 | 新记忆写入时自动生成 embedding，存入 IndexedDB 新字段 |
| 3 | 检索时语义排序 | 半天 | 候选集缩小 → 余弦相似度 top-5 → 失败自动回退索引排序 |
| 4 | 接入 `searchMemories()` | 0 | 接口无需改动，只需替换内部实现 |

### 架构要点

```
写入路径：新记忆 → 生成 embedding(Float32Array) → 存入 IndexedDB
检索路径：searchMemories() → 索引查询候选集(100条) → 向量相似度排序 → top-5
失败回退：模型未就绪/unload → 走纯索引+重要性+时间排序（效果不变）
```

**设计原则：** 语义搜索是插拔式的，不会引入任何新的外部依赖风险。加载失败、加载慢、模型被清理等场景都有降级路径。

### 触发条件

- 优先：在 v1 完成后立刻启动
- 最低：任一用户记忆条数 ≥ 100 条前完成

## 快速回滚说明

### 恢复骨架文件（紧急情况）
如果旧骨架文件需要临时恢复，在 index.html 中添加对应的 script 标签即可：
```html
  <script src="js模块/记忆常量.js"></script>
  <script src="js模块/记忆实体.js"></script>
  <script src="js模块/记忆索引.js"></script>
  <script src="js模块/记忆管理器.js"></script>
  <script src="js模块/记忆接口.js"></script>
  <script src="js模块/记忆检索.js"></script>
```

### 回滚注入策略
如果记忆检索注入导致异常，注释 `对话管理.js` 中的注入代码块（以 `// 2.7 主动记忆检索注入` 搜索定位）即可。

### 回滚 searchMemories
旧的 `搜索()` 方法保留为 `searchMemories` 的代理，两者兼容。如需完全回退到旧逻辑，将 `对话管理.js` 中的 `searchMemories` 调用改回 `搜索` 即可。
